# RISC_V-MULTICYCLE-PROCESSOR_-PRELIMINARY
This repository is different from the one submitted in the homework HW2, which was all about the controller , this is the whole RISC-V Multicycle Processor, this one uses the usual encoding layout of the controller given in the HW2.
Note: In the HW2 repository, since the original assignment required no changes to the testbench code and testvectors, the aludec module (ALU Decoder) was adjusted for it, this is NOT the case for this ALU Decoder as it uses the given encoding in HW2. Therefore, the aludec modules in these assignment are different.
